# Skill: Strategie-Framework-Toolkit (Unternehmensberater)

Ein Framework ist nur so gut wie seine Passung zur Frage. Dieser Skill **waehlt das richtige** (1-2, nicht
alle) und wendet es sauber an -- die Substanz zaehlt, nicht das Framework-Feuerwerk.

## Wann anwenden
Bei strategischen Fragen: Positionierung, Wettbewerb, Wachstum, Portfolio/Ressourcen, Geschaeftsmodell.

## Framework -> wann
- **SWOT** -- schneller Lage-Ueberblick (Staerken/Schwaechen intern; Chancen/Risiken extern). Guter Startpunkt,
  aber flach -> nicht als alleinige Analyse.
- **Porter's Five Forces** -- Attraktivitaet/Wettbewerb eines Marktes (neue Anbieter, Ersatz, Lieferanten-/
  Kundenmacht, Rivalitaet). Fuer „lohnt sich dieser Markt/diese Nische?".
- **BCG-Matrix (Growth-Share)** -- **Portfolio/Ressourcenverteilung** ueber Produkte/Einnahmequellen (Star /
  Cash Cow / Question Mark / Poor Dog): worein investieren, was halten, was einstellen.
- **Ansoff-Matrix** -- **Wachstumsoptionen** (Marktdurchdringung · Marktentwicklung · Produktentwicklung ·
  Diversifikation) und ihr Risiko.
- **Business Model Canvas** -- ganzes **Geschaeftsmodell**/neue Idee auf einen Blick (Wertangebot, Segmente,
  Kanaele, Einnahmen, Kosten ...).
- **Wertkette** -- interne **Effizienz/Wertschoepfung** (wo entsteht Wert, wo Verlust?).

## Vorgehen
1. Frage-Typ bestimmen -> **1-2 passende** Frameworks waehlen (Begruendung der Wahl).
2. Framework mit **echten Daten** fuellen (CDO-Zahlen, Research) -- keine Platzhalter.
3. **So what?** ableiten: was folgt konkret daraus (nicht nur die Matrix zeigen).
4. Ergebnis in `strukturiertes-problemloesen` einbetten (Pyramid-Output).

## Grenzen
Analyse-Grundlage, keine Entscheidung. Frameworks sind Hilfsmittel -- wenn keins passt, **einfach klar
argumentieren** statt eins erzwingen.
